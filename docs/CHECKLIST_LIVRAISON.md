# Checklist finale de livraison — Artisans BF

## GitHub

- [ ] Dépôt public ou accessible au jury.
- [ ] README complet.
- [ ] Branches propres : `main`, `develop`, `feature/*`.
- [ ] PR utilisées pour intégrer le travail.
- [ ] `.gitignore` présent.
- [ ] Aucun fichier `.env` envoyé.
- [ ] Aucune base locale lourde envoyée.
- [ ] Workflows GitHub Actions présents.

## Backend

- [ ] Installation avec `pip install -r requirements.txt`.
- [ ] Lancement avec `uvicorn app.main:app --reload`.
- [ ] Swagger accessible sur `/docs`.
- [ ] Authentification fonctionnelle.
- [ ] Commerces fonctionnels.
- [ ] Photos limitées à 1–3.
- [ ] Publication/retrait fonctionnels.
- [ ] Recherche et distance fonctionnelles.
- [ ] Commentaires et notation IA fonctionnels.

## Frontend

- [ ] Accueil accessible.
- [ ] Inscription et connexion accessibles.
- [ ] Tableau de bord accessible.
- [ ] Ajout commerce accessible.
- [ ] Recherche publique accessible.
- [ ] Carte visible.
- [ ] Détail commerce accessible.
- [ ] Boutons Appeler, WhatsApp, Itinéraire fonctionnels.

## Documentation

- [ ] Documentation technique PDF créée.
- [ ] Moins de 20 pages.
- [ ] Moins de 10 Mo.
- [ ] Lien GitHub présent.
- [ ] Procédure de déploiement présente.
- [ ] Architecture présente.
- [ ] Membres et coordonnées présents.
- [ ] Captures principales présentes.

## Livraison

- [ ] ZIP final inférieur à 100 Mo.
- [ ] ZIP testé sur une autre machine.
- [ ] Données de démonstration présentes.
- [ ] Scénario jury répété au moins une fois.
